### M2 — SIGTERM drain test (1h)

Closes the ⚠️ partial in [plan section 8 item 2](../crash-recovery-workflow-resilience-plan.md). The chaos test used SIGKILL; this one uses SIGTERM to specifically exercise T1.3 graceful shutdown.

```powershell
# Submit 5 orders one after another (don't wait between submits)
1..5 | ForEach-Object {
    $idx = $_
    $orderRef = "SIGTERM-$([guid]::NewGuid().ToString('N').Substring(0,8))-$idx"
    $body = @{
        SourceSystem = "Oms"
        OrderRef = $orderRef
        Priority = "Normal"
        RequestedTransportMode = "Amr"
        ServiceWindow = @{ EarliestUtc = $null; LatestUtc = (Get-Date).AddHours(4).ToUniversalTime().ToString("o") }
        Items = @(@{
            ItemId = "ITEM-$([guid]::NewGuid().ToString('N').Substring(0,8))"
            PickupLocationCode = "2138213c-f7eb-40d8-a6f7-0ab435012f4d"
            DropLocationCode = "565ccc96-329c-4113-956d-a0d1aa1ed786"
            LoadUnitProfileCode = "PALLET-EU"
            WeightKg = 5.0
            Quantity = @{ Value = 1.0; Uom = "BOX" }
        })
    } | ConvertTo-Json -Depth 5

    $req = New-Object System.Net.Http.HttpRequestMessage("POST", "http://localhost:5219/api/v1/delivery-orders/upstream")
    $req.Content = [System.Net.Http.StringContent]::new($body, [System.Text.Encoding]::UTF8, "application/json")
    $req.Headers.Add("Idempotency-Key", "sigterm-$orderRef")
    $client = New-Object System.Net.Http.HttpClient
    $client.SendAsync($req).Result.StatusCode
}

# Within 2-3 seconds of submitting the 5th, SIGTERM the api
Start-Sleep -Seconds 2
$killStart = Get-Date
docker kill --signal=SIGTERM dtms-api
# Wait until container exits, time it
while ((docker ps --format "{{.Names}}" | Select-String "^dtms-api$")) { Start-Sleep -Seconds 1 }
$killEnd = Get-Date
"Pod exit took $([int]($killEnd - $killStart).TotalSeconds)s"

# Restart
docker compose up -d api
Start-Sleep -Seconds 30  # let it boot

# After 3-min settle, verify the 5 reached Dispatched
Start-Sleep -Seconds 180
docker exec dtms-postgres psql -U postgres -d amr_delivery_planning --% -c "SELECT \"OrderRef\", \"Status\" FROM deliveryorder.\"DeliveryOrders\" WHERE \"OrderRef\" LIKE 'SIGTERM-%' ORDER BY \"CreatedDate\" DESC LIMIT 5;"
```

**Accept**:
- Pod exit ≤ 90s
- All 5 SIGTERM orders end at `Dispatched` (or terminal)
- No order at `Planned` post-settle

### A1. Add `During()` handlers for redelivery dedup (30 min)

In `src/Modules/Planning/DTMS.Planning.Infrastructure/Sagas/DeliveryOrderSagaStateMachine.cs`, after the `Initially(…)` block, add:

```csharp
// T2 Phase 2 step 1 — redelivery dedup. After the saga has entered any
// user-defined state, an OrderConfirmed redelivery (T1.1 retry, T1.4
// watchdog, T1.7 admin /replan) is a no-op rather than a fault. Without
// this, MassTransit raises NotAcceptedStateMachineException and the
// message goes to DLQ.
During(AwaitingPlan,        Ignore(OrderConfirmed));
During(Planning,            Ignore(OrderConfirmed));
During(Dispatching,         Ignore(OrderConfirmed));
During(Completed,           Ignore(OrderConfirmed));
During(FailedAwaitingRetry, Ignore(OrderConfirmed));
```

Build + run unit tests (none exist for the saga yet — add a smoke test in `tests/Modules/Planning.UnitTests` that constructs the state machine and asserts compile + bind).

### A2. Add the first real transition: `OnPlanRequested` → Planning (1h)

The current state machine has Initially → AwaitingPlan and stops. Add the next step:

1. Decide where `PlanRequested` comes from. Options:
   - From the legacy `DeliveryOrderValidatedConsumer` after `MarkOrderPlanning` succeeds (cleanest for dual-run)
   - From a new event the saga itself raises (not yet — requires more wiring)

   **Pick option 1.** Reuses what's already there.

2. Define the event in `Planning.IntegrationEvents`:

   ```csharp
   public record OrderPlanRequestedIntegrationEventV1(
       Guid EventId,
       DateTime OccurredOn,
       Guid DeliveryOrderId,
       string SchemaVersion = "1.0") : IIntegrationEvent;
   ```

3. Publish it from the legacy consumer after `MarkOrderPlanning` (in `DeliveryOrderValidatedConsumer.cs`):

   ```csharp
   await _sender.Send(new MarkOrderPlanningCommand(evt.DeliveryOrderId), ct);
   await context.Publish(new OrderPlanRequestedIntegrationEventV1(
       Guid.NewGuid(), DateTime.UtcNow, evt.DeliveryOrderId), ct);
   ```

4. Wire it into the saga:

   ```csharp
   Event(() => PlanRequested,
       e => e.CorrelateById(ctx => ctx.Message.DeliveryOrderId));

   During(AwaitingPlan,
       Ignore(OrderConfirmed),
       When(PlanRequested)
           .Then(ctx => { ctx.Saga.UpdatedAtUtc = DateTime.UtcNow; })
           .TransitionTo(Planning));
   ```
